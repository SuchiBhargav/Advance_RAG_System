What is st.session_state?

    st.session_state is a special dictionary-like object provided by Streamlit.

    It stores values that persist across reruns of your Streamlit app.

    Normally, Streamlit runs your script top to bottom on every user interaction, so all variables reset.

    st.session_state lets you keep data alive between those reruns — like user inputs, flags, counters, etc.

How does it work internally?

    Streamlit runs your entire script on every interaction (like a button click or input).

    Before each run, Streamlit restores st.session_state from the browser/user session.

    When your script modifies st.session_state, Streamlit saves those changes back to the session.

    This way, data inside st.session_state persists across reruns and between widgets.
