For Dell nodes, the image mounting process is different. An HTTP service is being used to offer the image. Therefore, this step may be failing because the node cannot connect to the HTTP service. For this failure, we can check three things:

1 The http service must be active on Mgen's laptop

[root@roadieag7 logs]# service httpd status 

Redirecting to /bin/systemctl status httpd.service

● httpd.service - The Apache HTTP Server

   Active: active (running) since Sat 2025-03-22 22:01:10 EDT; 1 months 17 days ago

     Docs: man:httpd.service(8)

2 Make sure the port is open in case the firewalld is active

firewall-cmd --permanent --add-port=8118/tcp
